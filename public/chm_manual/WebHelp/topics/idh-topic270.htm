2.  Đơn vị tính trọng số của lỗi. Phân loại weighted defect như sau:

Bug Weight

Description

Low (1)

Lỗi rất nhỏ và không ảnh hưởng gì đến tính năng của sản phẩm: GUI, grammar, message

Medium (2)

Tính năng nhỏ không làm việc được hoặc làm không đúng hoặc ghi chép thành tài liệu không đúng

High (3)

Tính năng lớn hoặc 1 thành phần của hệ thống không làm việc hoặc làm việc không đúng hoặc ghi chép thành tài liệu không đúng

Serious (5)

Tính năng lớn quan trọng của hệ thống không làm việc

 

Các yêu cầu về đặc tính chất lượng không thỏa mãn như:  performance effects, security problem

Fatal (8)

Lỗi fatal khi mà phần lớn tính năng hoặc các thành phần rất quan trọng của hệ thống bị sập. Việc thực hiện hoặc test không thể tiếp diễn khi xuất hiện lỗi này

 

3. Bug Type - loại lỗi

 

Name

Description 

GUI

Lỗi về giao diện, text, message, grammar, syntax

FUN

Lỗi về tính năng

API

Lỗi liên kết với các hệ thống khác

Other

Các lỗi khác

 

4. Bug root cause - Nguyên nhân gốc rễ của lỗi

 

Name

Description 

Requirements

Lỗi gây ra bởi yêu cầu không đầy đủ hoặc không rõ ràng

Design Error

Thiết kế sai khác với yêu cầu đưa ra hoặc do thiết kế không rõ ràng, không đầy đủ

Code Error

Code sai khác với tài liệu thiết kế hoặc yêu cầu đưa ra, hoặc là code sai về cú pháp, cấu trúc bên trong

Test Error

Test được thiết kế không đúng ( sai khác với yêu cầu hoặc thiết kế) hoặc các bước chạy sai, cấu hình sai, nhập dữ liệu sai dẫn đến đăng lỗi sai

Configuration

Lỗi gây ra do môi trường hoặc dữ liệu cấu hình không đúng

Existing bug

Lỗi đã xuất hiện trước đây và giờ lại xuất hiện lại
